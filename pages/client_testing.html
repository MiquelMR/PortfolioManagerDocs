Testing on the client side
# Logging In
The user attempts to log in from the login screen
Condition Result
_service:ActiveUser contains the user object Success
_localStorage.Token contains the user token Success
_localStorage.Email contains the user email Success
Navigates to Home page automatically Success
The homepage navbar displays the user's avatar and name Success
# Logging Out
The user logs out
Condition Result
_service:ActiveUser is null Success
_localStorage.Token is null Success
_localStorage.Email is null Success
Navigates to Login page automatically Success
# Registering an Account
The user attempts to register in the app
Condition Result
_service:ActiveUser contains the user object Success
_localStorage.Token contains the user token Success
_localStorage.Email contains the user email Success
Navigates to Home page automatically Success
An unsuccessful registration does not navigate to the Home page automatically Success
The new user is able to log in Success
he user's avatar is stored on the server Success
Validation: Name can't be empty Success
Validation: Avatar can be empty Success
Validation: Email can't already exist Success
# Deleting an Account
The user attempts to delete their account
Condition Result
_service:ActiveUser is null Success
_localStorage.Token is null Success
_localStorage.Email is null Success
Navigates to Login page automatically Success
The user's avatar is deleted from the server Success
The user is deleted in the database Success
# Updating an Account
The user attempts to update their Account
Condition Result
Navigates to Login page automatically Success

The old avatar, if available, is deleted from the server Success
