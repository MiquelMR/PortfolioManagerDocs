# Data layer : Services and Repositories
- Services and repositories follow the same naming ruleset
- Methods should appear in the order defined here
- The order of appearance in the interface and inherited class should be the same
## Method naming ruleset

Operation on a single entity : OperationVerbEntity

Example : UpdateUser

Operation on a single entity with filter : OperationVerbEntityByProperty

Example : GetUserByUserId

Operation on multiple entities : OperationVerbEntities

Example : GetUsers

Utility Methods : OperationVerbByProperty

Example : ExistsByEmail

Methods should appear in the following order

  1. Get single entity
  2. Get multiple entities
  3. Get multiple entities by filter
  4. Create
  5. Update
  6. Delete
  7. Public Utilities
  8. Private Utilities
